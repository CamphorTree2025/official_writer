## Bug 修复：模板管理页面无法加载

- **根本原因**：`templates.html`、`config.html`、`generate.html` 三个页面的 `{% block scripts %}{% raw %}` 之后缺少 `<script>` 标签，只有 `</script>` 关闭标签，导致浏览器将 JS 代码当作 HTML 文本渲染而非执行
- **修复方式**：在三个文件的 `{% raw %}` 后添加 `<script>` 标签
- **教训**：Jinja2 的 `{% raw %}` 只是禁止 Jinja2 解析花括号，不会自动生成 `<script>` 标签

## 变量顺序修复：保持模板出现顺序

- **问题**：预览编辑页面的变量显示顺序与模板中的顺序不一致
- **根因**：`template_parser.py` 的 `parse()` 和 `get_preview()` 方法使用了 `sorted(list(variables))`，按字母排序丢失了模板出现顺序
- **修复**：将 `set()` + `sorted()` 改为 `seen` + `ordered_variables` 列表，按段落→表格遍历顺序保持变量出现顺序
- **同步修复**：`variable_engine.py` 的 `fill_template()` 返回值按 `variables` 列表顺序构建 dict，确保 JSON 序列化后前端 `Object.entries()` 也保持正确顺序

## 填充模式双模式改造

- **需求**：用户要求两种填充模式——严格提取（不改原文）和 AI 润色（自动补充）
- **VariableEngine 改造**：`fill_template()` 新增 `mode` 参数（strict/polish），返回 `(filled_content, sources)` 元组
  - strict 模式：System Prompt 指示只从输入中提取原文，未识别输出 `[未识别]`，后端转为空字符串+manual标记
  - polish 模式：System Prompt 指示润色+补充缺失内容
  - sources 字典：`extracted`（原文提取）/ `polished`（AI润色）/ `manual`（需手动填写）
- **后端改造**：`/api/preview` 接受 `mode` 参数，返回 `variables` + `sources` + `mode`
- **前端改造**：
  - 步骤2新增模式选择卡片（📌严格提取 / ✨AI润色），默认选中润色模式
